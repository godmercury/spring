1. Oracle
    관계형 데이터베이스 관리 시스템(RDBMS)
    기업용으로 주로 사용
    대량의 정보관리를 할 때 타 DBMS에 비해 좋은 성능을 보인다


2. Mysql
    관계형 데이터베이스 관리 시스템(RDBMS)
    오픈 소스이며, 다중 사용자와 다중 스레드 지원


3. MS-SQL
    관계형 데이터베이스 관리 시스템(RDBMS)
    분산된 기업환경에서의 집둥된 서버 관리 도구 제공
    중앙 집중식 데이터베이스 제어로 통신 누락, 오류 발생 최소화


4. PostrgreSQL
    오픈 소스 객체-관개형 데이터베이스 시스템(ORDBMS)
    북미나 일본에서 많이 사용
    다양한 데이터베이스 객체를 사용자가 임의로 만들 수 있는 기능을 SQL 차원에서 제공
    테이블 상속 기능을 이용해 하위 테이블 생성 가능
    기본적인 CRUD 성능이 경쟁 DB에 비해 좋지 않다


5. MongoDB
    크로스 플랫폼 도큐먼트 지향 데이터베이스 시스템
    NoSQL 데이터베이스
    모든 데이터가 JSON 형태로 저장, 스키마가 없음
    다양한 인텍싱 제공
    일관성이 매우 중요한 작업에는 사용하기 힘들다


6. Redis
    Key-Value 구조의 비정형 데이터를 저장하고 관리하기 위한 오픈 소스 기반의 비 관계형 데이터베이스 관리 시스템
    모든 데이터를 메모리로 불러와서 처리하는 메모리기반 데이터베이스 시스템
    메모리 기반이므로 속도가 매우 빠르다
    문자열, 리스트, 해시, 셋, 정렬된 셋 같은 다양한 자료구조의 지원


7. SQLite
    독립형 파일 기반의 오픈소스 관계형 데이터베이스 관리 시스템(RDBMS)
    매우 가볍고 이식성이 뛰어남
    완전 독립형이라 SQLite를 작동하기 위해 시스템에 설치해야 하는 외부 종속성이 없음
    동시성의 제한, 사용자 관리 존재의 부재, 서버리스 데이터베이스이기에 보안이 약함


8. MariaDB
    오픈소스 관계형 데이터베이스 관리 시스템(RDBMS)
    Mysql과 거의 100% 호환성을 가지고 있음
    MySQL에 비해 애플리케이션 부분 속도가 약 4~5천배 빠르고, 성능면에서는 70% 향상을 보인다고 함

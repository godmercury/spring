Protocol : 통신 프로토콜 혹은 통신 규약. 컴퓨터나 원거리 통신 장비 사이에서 메시지를 주고 받는 양식과 규칙.

기본요소
 - 구문(Syntax) : 전송하고자 하는 데이터의 형식(Format), 부호화(Coding), 신호 레벨(Signal Level) 등을 규정
 - 의미(Semantics) : 두 기기 간의 효율적이고 정확한 정보 전송을 위한 협조 사항과 오류 관리를 위한 제어 정보를 규정
 - 시간(Timing) : 송/수신자간 혹은 양단(End-to-End)의 통신 시스템과 망 사이의 통신 속도나 순서 등을 정의



기능
 - 단편화(Fragmentation)와 재합성(Assembly) : 송신측에서 긴 데이터를 블럭으로 쪼개고 수신측에서 쪼개진 블럭을 재합성해서 원래의 데이터 복원
 - 캡슐화(Encapsulation) : 플래그, 주소, 제어정보, 오류 검출 부호 등을 부착하여 프로토콜에 적합한 데이터 블록을 생성
 - 연결제어(Connection Control) : 비연결 데이터 전송(데이터그램)과 연결 위주 데이터 전송(가상회선)을 위한 통신로를 개설,유지,종결하는 기능
 - 흐름제어(Flow Control) : 데이터 양이나 통신 속도 등이 수신 측의 처리 능력을 초과하지 않도록 조정하는 기능
 - 오류제어(Error Control) : 데이터 전송 중 발생할 수 있는 오류나 착오 등을 검출하고 정정하는 기능
 - 순서결정(Sequencing) : 연결 위주의 데이터를 전송할 때 송신 측이 보내는 데이터 단위 순서대로 수신측에 전달하는 기능
 - 주소설정(Addressing) : 발생지, 목적지 등의 주소를 명기하여 데이터를 정확하게 전달하는 기능
 - 동기화(Synchronization) : 두 통신 객체의 상태(시작, 종류, 검사 등)를 일치시키는 기능
 - 다중화(Multiplexing) : 하나의 통신로를 여러 개로 나누거나 회선 여러 개를 하나의 통신로로 변환하여 다수의 가입자가 동시에 사용할 수 있도록 하는 기능
 - 전송서비스(Transmission Service) : 통신 객체를 사용하기 쉽도록 별도로 추가 서비스(패리티 검사, 보안도, 서비스 등급, 우선 순위 등)를 제공하는 기능


종류
7. 응용계층(Application) - 전송단위: Data
    사용자가 볼 수 있는 유일한 계층으로 모든 네트워크 활동의 기반이 되는 인터페이스를 제공
    - HTTP (HyperText Transfer Protocol) : 웹(응용계층)에서 하이퍼텍스트(HTML) 문서를 교환하기 위해 사용되는 프로토콜. port 80
    - FTP (File Transfer Protocol) : 응용계층에서 파일을 주고 받기 위한 규약. port 20(데이터 전송용), 21(제어용)
    - Telnet: 인터넷을 통하여 원격지의 호스트 컴퓨터에 접속할 때 지원되는 인터넷 표준 프로토콜
    - SSH (Secure Shell) : 원격 접속시 지원되는 인터넷 표준 프로토콜로 보안을 중시함
    - SMTP (Simple Mail Transfer Protocol) : 사용자의 컴퓨터에서 작성된 메일을 받아서 다른 사람의 계정이 있는곳으로 전송해 주는 전자 우편 송신 프로토콜. port 25
    - POP3 (Post Office Protocol) : 전송받은 메일을 저장하고 있다가 사용자가 메일 서버에 접속하면 이를 보내주는 역할을 하는 전자 우편 수신 프로토콜. port 110

6. 표현계층(Presentation) - 전송단위: Data
    응용 계층으로부터 전달받은 데이터를 읽을 수 있는 형식으로 변환. 
    응용계층으로부터 전송받거나 응용계층으로 전달해야 할 데이터의 인코딩과 디코딩이 이루어지고, 데이터를 안전하게 사용하기 위해 암호화와 복호화를 한다.
    - SSL (Secure Socket Layer) : 보안 소켓 계층. 네트워크 인증, 암호화 등의 규약
    - TLS (Transport Layer Security) : 전송 계층 보안. 인터넷에서의 정보를 암호화해서 송수신하는 프로토콜
    - JPEG
    - MPEG

5. 세션계층(Session) - 전송단위: Data
    데이터가 통신하기 위한 논리적 연결을 담당
    두 컴퓨터 간의 대화나 세션을 관리하며, 포트 연결이라고도 한다.
    모든 통신 장치 간에 연결을 설정하고 관리하고 종료한다.
    - PPTP
    - NetBIOS

4. 전송계층(Transport) - 장비: 게이트웨이 (전송단위: Segments)
    TCP와 UDP 프로토콜을 통해 통신을 활성화 한다.
    포트를 열어두고, 프로그램들이 전송을 할 수 있도록 제공
    - TCP (Transmission Control Protocol) : 전송 계층에서 정보 전달을 안전하게 할 수 있도록 하는 규약. 신뢰성, 연결지향적
    - UDP (User Datagram Protocol) : 전송 계층에서 보다 빠르게 데이터를 주고 받을 수 있도록 하는 규약. 비신뢰성, 비연결성, 실시간

3. 네트워크 계층(Network) - 장비: 라우터 (전송단위: Packets)
    데이터를 목적지까지 가장 안전하고 빠르게 전달하는 기능
    라우터를 통해 이동할 경로를 선택하여 IP 주소를 지정하고, 해당 경로에 따라 패킷을 전달한다.
    라우팅, 흐름제어, 오류제어, 세그먼테이션 등을 수행
    - IP (Internal Protocol) : 네트워크 계층에서 호스트의 주소 지정과 패킷을 분할하고 조립하는 규약
    - ICMP
    - ARP

2. 데이터링크계층(Data Link) - 장비: 브릿지, 스위치 (전송단위: Frames)
    물리 계층으로 송수신되는 정보를 관리하여 안전하게 전달되도록 도와주는 역할로 Mac 주소를 통해 통신
    프레임에 Mac 주소를 부여하고 에러검출, 재전송, 흐름전송을 진행
    - MAC
    - PPP
    - ARP
    - Ethernet

1. 물리계층(Physical) - 장비: 허브, 리피터, 케이블, 모뎀 (전송단위: Bits)
    데이터를 전기적인 신호로 변환해서 주고받는 기능을 진행. 데이터를 전송하는 역할만 진행한다.
    - Ethernet
    - USB
    - Bluetooth
    - IEEE802.11




 ex. 미국에 있는 친구에게 편지를 보낸다
 응용: 편지를 쓴다
 표현: 한글로 작성한 편지를 미국 친구가 알아볼 수 있게 번역한다
 세선: 미국 친구의 집 주소를 기입한다
 전송: 우체국에 편지를 접수시키기 위한 절차, 즉 배 또는 비행기 등의 운송수단 결정
 네트워크: 우체국에 있는 여러 편지들을 같은 목적지별로 분류한다
 데이터링크: 해당되는 목적지와 운송 방법에 따라 분류하며, 해당되는 목적지와 직접 연결이 되지 않는 경우 중간 경유지를 선택하여 분류하는 작업
 물리: 실제적으로 편지가 배, 비행기 등의 운송 수단에 의해 운송되는 것을 의미

 ex.  이메일 전송
 응용: 이메일 프로그램을 통해 이메일 작성
 표현: 공통된 표현 방식으로 데이터를 변환하거나, 암호화, 합축 수행
 세션: 데이터의 동기화를 위해 일정한 길이마다 sync를 삽입하여 전송 계층으로 데이터 전달
 전송: 발신지와 목적지의 주소를 지정하고, 연결 방식, 흐름 제어 오류 제어를 한다. 그리고 데이터를 전송할 수 있는 세그먼트 단위로 나눈다
 네트워크: 발산지와 목적지의 주소가 아닌 라우팅에 필요한 논리 주소를 설정하고, 패킷에 대한 라우팅 정보 삽입
 데이터링크: 프레임 단위로 데이터를 나눈다. Mac 주소를 지정하고 각 양 끝단의 속도 차이에 대해 월활하게 해주기 위해 흐름 제어를 한다. 또한 데이터의 오류를 막기 위해 CRC 방법이나 체크썸 방식을 사용해 데이터를 받는 쪽에서 데이터의 오류를 검사할 수 있도록 한다.
 물리: 전송 매체가 일반 케이블인지, 광 케이블인지 등의 설정을 한다. 그리고 전송 방식과 데이터를 회선으로 보내기 위한 전기적인 변환을 담당한다.



공유기
하나의 공인IP를 사용하여 여러 개의 기기가 인터넷을 사용하기 위해 사용되는 네트워크 기기 (소용량의 라우터)


허브
여러 대의 컴퓨터를 연결해서 네트워크를 만들어주는 장치. 
데이터 패킷을 받으면 연결된 모든 장치들에게 모두 보낸다. 모든 장치들이 데이터 패킷을 받고 자신에게 온 데이터 패킷인지 확인하고 필요하면 취득한다. 즉, 단순 분배를 하는 중계 장치. 
허브는 모든 장치에게 패킷을 뿌리기 때문에 자신이 가지고 있는 데이터 전송 대역을 분리해서 사용해야 한다. (허브의 데이터 전송 속도가 100Mbps이고 연결 장치가 5대이면 각각의 대역폭은 20Mbps)
허브는 IP 할당 기능은 따로 있지 않고 리피터와 멀티포트의 역할을 한다.


스위치
연결된 장치들의 IP와 MAC 주소를 모두 테이블 형태로 가지고 있고, 원하는 목적지에 데이터 패킷을 전송하는 장치
IP와 MAC 주소를 기반으로 필요한 장치에게만 데이터 패킷을 전송해준다.
IP와 Port가 일치하지 않는 MAC 주소가 테이블에 존재하지 않는 경우 허브처럼 모든 장치에게 데이터를 뿌린다.(포워딩)
허브와는 달리 데이터 전송 대역을 분리하지 않는다. (스위치의 데이터 전송 속도가 100Mbps이고 연결 장치가 5대이면 각각의 대역폭은 100Mbps)


라우터
LAN(Local Area Network: 근거리 통신망)을 연결해주는 장치
데이터를 전송할 때 최종 목적지에 가기 위하여 거쳐야 하는 많은 경로들 중 적절한 통신 경로를 결정하기 위해 한 통신망에서 다른 통신망으로 데이터 패킷을 전송하는 장치


게이트웨이
서로 다른 통신망, 프로토콜을 사용하는 네트워크 간의 통신을 가능하게 하는 기기
일종의 번역기 혹은 환전소 
ex. 이동통신 장비와 인터넷 간의 게이트웨이


브릿지
허브의 기능을 가지며 데이터를 보낼 때 Mac Address라는 고유 주소를 가지고 통신
스위치와 전송 방법이 다르다.
오늘날에는 순수 브릿지는 거의 사용 안함





리피터(Repeater) : 전송 거리가 짧아서 신호가 도달하지 못하는 경우에 설치하여 신호전달을 용이하게 합니다. 

멀티포트(Multi-Port) : 공유기의 포트가 모자란 경우 설치하여 포트를 늘리는 것 

ISP(Internet Service Provider) : 인터넷 서비스 제공자는 인터넷에 접속하는 수단을 제공하는 주체
